03 · Architecture Technique
Une seule base de code,
un écosystème complet.
Flutter (Dart)
Framework cross-platform. Une base de code unique pour Android et Flutter Web. Style Luxury Noir & Bleu & Or, animations au scroll.
Firebase Firestore
Base de données NoSQL temps réel. Streams pour messagerie et dons. Règles de sécurité strictes par rôle.
Firebase Auth
Authentification email, Google Sign-In et SMS (OTP). Redirection automatique selon le rôle Firestore.
OpenStreetMap
Cartographie 100% gratuite via flutter_map + latlong2. Géolocalisation avec geolocator. Zéro carte bancaire.
Firebase Storage
Stockage des images et fichiers de la messagerie. Règles de sécurité : 10 MB max par fichier.
Firebase Hosting
Déploiement Flutter Web + Landing page React. URL publique HTTPS. PWA installable sur mobile.

04 · Fonctions Clés — Système de Dons
Un don transparent,
validé avant d'être comptabilisé.
💳
Bailleur fait un don
Choisit un montant en FCFA (prédefini ou libre), sélectionne la méthode de paiement (Mobile Money, carte, virement). Statut : EN ATTENTE.
STATUS: PENDING
Message automatique
Un message est envoyé automatiquement à l'ONG dans la messagerie : montant, programme concerné, message du bailleur.
CHAT AUTO-CRÉÉ
ONG confirme
L'ONG voit le don dans son tableau de bord et clique CONFIRMER. Le montant est alors ajouté au programme. Message de confirmation envoyé au bailleur.
STATUS: CONFIRMED
Tant que le don n'est pas confirmé par l'ONG...
Le montant n'est pas ajouté au programme. L'historique du bailleur ne comptabilise que les dons confirmés. Transparence totale.
VALIDATION ONG
Ce que tu dis
1
Le problème que ça résout : "Dans beaucoup de plateformes, le don est affiché immédiatement même si l'ONG ne l'a pas reçu. J'ai voulu éviter ça. Ici, le don est EN ATTENTE jusqu'à confirmation explicite de l'ONG."
2
Techniquement : "J'utilise un batch Firestore : le don est créé avec status 'pending', et un message automatique est envoyé dans le chat ONG-Bailleur. Quand l'ONG confirme, le batch met à jour le statut ET incrémente raisedAmount du programme en une seule transaction atomique."
3
L'UX bailleur : "Le bailleur reçoit une notification push quand son don est confirmé. Et l'historique de ses dons ne montre que les montants réellement validés."

Ce que tu dis
1
Le principe : "J'ai volontairement limité les pouvoirs de l'admin. Il voit tout, mais ne peut rien modifier. Sa seule capacité d'action : la messagerie pour contacter n'importe quel utilisateur."
2
Pourquoi ce choix : "Un admin qui peut tout modifier est un risque. Un admin qui supervise et communique est un outil de qualité. C'est un choix délibéré de conception."

Techniquement : "Le rôle admin est stocké dans Firestore. Les règles de sécurité Firebase vérifient le rôle avant chaque lecture. Il est impossible de contourner ça côté client."
